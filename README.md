# Управление регистрацией данных

Расширение 1С для гибкого управления регистрацией объектов при обмене данными через универсальный формат.

---

## Назначение

Стандартная подсистема «Обмен данными» (БСП) определяет правила регистрации объектов через макет плана обмена или менеджер регистрации. Эти правила применяются одинаково ко всем узлам плана обмена - точечно управлять каждым узлом в отдельности не получается. Любое изменение логики регистрации требует доработки.

Данное расширение решает задачу иначе: встраивается в типовой механизм регистрации без изменения конфигурации, оставляя стандартные правила нетронутыми и полностью работоспособными после любых типовых обновлений.

Это расширение добавляет слой переопределяемых правил, который выполняется **после** стандартных правил регистрации и позволяет:

- **Включать или выключать** регистрацию конкретных объектов для каждого узла отдельно
- **Писать собственную логику фильтрации** на языке 1С прямо в карточке правила
- Управлять всем через удобную форму со списком объектов

**Для кого:** администраторы и разработчики, настраивающие обмен данными в конфигурациях на базе БСП с планом обмена «Синхронизация данных через универсальный формат». Можно использовать подключитьь и другой план обмена, главное чтобы он поддерживал типовую регистрацию.

---

## Преимущества

- **Не требует доработки конфигурации** — всё через расширение, обновления типовой не ломают настройки
- **Гибкое управление потоками данных** - для каждого узла-корреспондента можно тонко управлять объемом и условиями регистрации данных
- **Встроенный редактор кода** — с подсветкой синтаксиса, конструктором запросов и браузингом метаданных
- **Пользовательский код в безопасном режиме** — можно писать произвольную фильтрацию на BSL без риска для системы
- **Кэширование правил** — правила загружаются в параметры сеанса, производительность не страдает

---

## Требования

- Платформа 1С:Предприятие **8.3.18** и выше
- Библиотека стандартных подсистем (БСП) **3.0** и выше

## Установка

1. Скачайте файл расширения (`.cfe`) из раздела [Releases](../../releases)
2. Откройте конфигуратор базы данных
3. Меню **Конфигурация → Расширения конфигурации**
4. Нажмите **Добавить из файла** и выберите `.cfe`
5. Снимите флаг **«Безопасный режим»**
6. Нажмите **ОК** и примените изменения
7. Нажмите проверить применимость объектов и выберите соответствия заимствованным **типовым** объектам.

Если вы используете одно расширение для доработок, то смёржите его через **Сравнить, объединить с конфигурацией из файла**.

## Подключение узла к переопределяемому механизму

По умолчанию для всех узлов переопределение **выключено** — стандартные правила работают как прежде.

Чтобы включить расширенное управление для конкретного узла:

1. Откройте карточку нужного узла плана обмена, например «Синхронизация данных через универсальный формат»
2. Найдите команду **«Правила регистрации объектов»** (добавляется расширением)
3. В открывшейся форме установите флаг **«Использовать переопределение правил регистрации»**

> Когда переопределение включено, все объекты по умолчанию **доступны** для регистрации — нужно явно запретить те, которые не должны выгружаться.

## Настройка правил

### Обновление списка объектов

После включения переопределения расширение:

- Прочитает состав плана обмена
- Загрузит правила обработки данных (ПОД) из менеджера обмена
- Создаст строку правила для каждого объекта

Список нужно обновлять каждый раз командой **Обновить правила**, когда в конфигурации добавляются новые объекты в состав плана обмена или меняются ПОД. Расширение сохранит ранее заданные настройки.

### Включение и выключение объектов

В списке правил доступны следующие настройки для каждого объекта:

| Поле | Описание |
|------|----------|
| **Регистрация разрешена** | Объект будет регистрироваться для этого узла |
| **Применить собственное правило** | Включает выполнение пользовательского кода вместо простого флага |
| **Это ПОД** | Только чтение. Признак того, что объект из правил обработки данных (ПОД), а не просто из состава плана обмена |

### Групповые операции

Выделите несколько строк в списке и воспользуйтесь командами:

- **«Отметить активным»** — разрешить регистрацию для всех выбранных объектов
- **«Отметить неактивным»** — запретить регистрацию для всех выбранных объектов

## Пользовательский код

Для объектов, где простого флага недостаточно, можно написать собственную логику фильтрации.

### Как включить

1. Откройте карточку правила двойным кликом
2. Установите флаг **«Применить собственное правило»**
3. Нажмите кнопку **«Настроить правило»** — откроется редактор кода

### Доступные переменные

| Переменная | Тип | Описание |
|------------|-----|----------|
| `ДанныеИБ` | Объект / Ссылка | Сохраняемый объект или запись регистра |
| `Узел` | СсылкаНаОбъект | Узел плана обмена, для которого выполняется проверка |
| `УзлыПолучатели` | Массив | Список узлов получетелей, содержит подобранные узлы после выполнения типовых правил, но до выполнения переопределяемых |
| `ИсключаемыеУзлы` | Массив | Список узлов, который будет исключен из Улов-получателей выполнения переопределяемых правил для текущего объекта|
| `ФильтрПройден` | Булево | **Выходная переменная.** `Истина` — зарегистрировать, `Ложь` — не регистрировать. |
| `ДополнительныеПараметры` | Структура | Контекст операции (см. ниже) |

**Поля `ДополнительныеПараметры`:**

| Поле | Описание |
|------|----------|
| `ОбъектМетаданных` | Объект метаданных (например, `Метаданные.Справочники.Контрагенты`) |
| `ЭтоРегистр` | Булево — объект является регистром |
| `ЭтоУдалениеОбъекта` | Булево — объект помечается на удаление |
| `РежимЗаписи` | РежимЗаписиДокумента (только для документов) |
| `Замещение` | Булево — режим замещения (только для регистров) |

### Примеры

**Фильтрация по реквизиту:**
```bsl
ФильтрПройден = ДанныеИБ.ТипНоменклатуры = Перечисления.ТипыНоменклатуры.Услуга;
```

**Не регистрировать группы в справочнике Склады:**
```bsl
ФильтрПройден = ДанныеИБ.ЭтоГруппа = Ложь;
```


> Код выполняется в **безопасном режиме** — соблюдается стандарт вызова исполняемого кода через безопасный режим.

## Как это работает

```
Запись объекта
    ↓
Стандартные правила регистрации (список узлов-получателей)
    ↓
Переопределяемые правила расширения:
    для каждого узла в списке получателей:
        ├── переопределение выключено → узел остаётся
        ├── переопределение включено, правило запрещает → узел исключается
        ├── переопределение включено, правило разрешает → узел остаётся
        └── переопределение включено, собственный код → выполняется код, результат определяет судьбу узла
    ↓
Итоговый список получателей
```

## Архитектура (для разработчиков)

| Модуль / Объект | Назначение |
|-----------------|-----------|
| `ОбменДаннымиСобытия` (заимствование в расширение) | Точка входа: декораторы `&Перед` / `&После` процедуры регистрации |
| `ОбщегоНазначенияПереопределяемый` (заимствование в расширение) | Регистрация обработчиков параметров сеанса согласно БСП методолгии |
| `ПРО_ДатаКэшаПравилРегистрации` | Параметр сеанса для механизма обновления кэшированых данных |
| `ПРО_КэшПравилРегистрации`  | Параметр сеанса для хранения кэшированных данных |
| `ПРО_ОсновнаяРоль` | Условная роль расширения конфигурации. Предполгается, что у пользователя доступ только на чтение и установки основных механизмов подсистемы |
| `ПРО_УправлениеПравиламиРегистрации` | Основная бизнес-логика, служебный контракт API |
| `ПРО_УправлениеПравиламиРегистрацииСлужебный` | Управление кэшем, инициализация параметров сеанса |
| `ПРО_УправлениеПравиламиРегистрацииПовтИсп` | Повторно используемые значения (кэш правил) |
| РС `ПРО_НастройкиУзловРегистрации` | Настройки на уровне узла (мастер-переключатель) |
| РС `ПРО_ПереопределяемыеПравилаРегистрацииОбъектов` | Правила на уровне «узел × объект» |
| Обработка `ПРО_УправлениеРегистрациейДанныхКонсольКода` | Встроенный редактор BSL-кода, взят и доработан из репозитория https://github.com/salexdv/bsl_console |
