Справочная информация: Фильтры правил регистрации объектов

Назначение подсистемы

Подсистема «ПРО управление регистрацией данных» предназначена для настройки правил обмена данными между информационными базами без изменения основной конфигурации.

Типовой сценарий использования: при обмене через Конвертацию данных 3 в конфигурации уже настроены типовые правила регистрации объектов. Однако иногда эти правила работают не так, как требуется для конкретного узла обмена. Вместо того чтобы дорабатывать конфигурацию, вы можете переопределить правила прямо в режиме 1С:Предприятия.

Основные настройки

1. Настройка узла обмена

Для каждого узла обмена можно включить или отключить использование переопределений:

  • Использовать переопределение регистрации — если выключено, система работает по типовым правилам; если включено, применяются ваши настройки. По умолчанию всем объектам устанавливается запрещающее правило.

2. Фильтры правил регистрации объектов

Это основной регистр, где хранятся настройки правил обмена для конкретных объектов (справочников, документов, регистров).

Параметры фильтра

Узел информационной базы — узел плана обмена, для которого настраивается правило.

Объект регистрации — объект метаданных (справочник, документ и т.п.), для которого настраивается правило.

Настройки правила

Регистрация объекта разрешена — простой способ управления:

  • Включено — объект будет регистрироваться для обмена
  • Выключено — объект не будет регистрироваться

Это ПОД — справочное поле, показывающее источник правила:

  • Да — правило взято из настроек менеджера обмена через универсальный формат
  • Нет — правило взято из состава плана обмена (объект метаданных)

Применить собственное правило — включите, если нужна сложная логика вместо простого «да/нет»

Исполняемый код — программный код на языке 1С для реализации сложных условий регистрации (см. примеры ниже).

Пример исполняемого кода:

Если СтрНайти(ДанныеИБ.Наименование, "Владимир Владимирович Пу") > 0 Тогда
    ФильтрПройден = Истина;
КонецЕсли;

Как это работает

  1. Вы настраиваете правила для конкретного узла обмена
  2. При обмене данными система проверяет ваши переопределения
  3. Если правило найдено — оно применяется вместо типового
  4. Если правило не найдено и переопределение включено — объект не регистрируется
